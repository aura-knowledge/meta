# Structure And Drafting

Use for outlines, article structure, examples, visuals, first drafts, and article-package planning.

## Process

1. Start from reader outcome, not topic inventory.
2. Use the smallest viable structure that can defend the thesis.
3. Put the main claim early.
4. Make each section answer one reader question or support one claim.
5. Pair abstract points with concrete examples.
6. Use visuals only when they explain sequence, comparison, system structure, taxonomy, or evidence flow.

## Article Shape

Prefer:

- title
- deck or short promise
- thesis
- reader context
- main sections tied to claims
- examples or diagrams
- limits and cautions
- practical takeaway
- sources

## Output

Return an outline or draft with claim coverage, source placeholders, visual/example notes, and unresolved gaps.
